## go-lockfree
Проект на языке Go, посвященный реализации структур данных без использования блокировок (lock-free). 
## О проекте
В отличие от классических структур данных, использующих мьютексы (sync.Mutex), lock-free алгоритмы позволяют горутинам работать с данными одновременно, не заставляя друг друга ждать. Это достигается за счет использования CAS (Compare-And-Swap) операций.
## Реализованные алгоритмы## 1. Treiber Stack (Стек Трайбера)
Классический lock-free стек на базе связного списка.

* Файл: internal/stack/treiber.go
* Принцип работы: использует atomic.Pointer для управления вершиной стека (head).
* Безопасность: за счет CAS-цикла гарантируется, что ни один узел не будет перезаписан или потерян при одновременном доступе из сотен горутин.

## Как это работает (на примере Push)
Алгоритм добавления элемента выполняется в три этапа:

   1. Снимок: Горутина делает локальную копию текущей вершины стека.
   2. Подготовка: Создается новый узел, который ссылается на этот "снимок".
   3. Фиксация (CAS): Процессор атомарно проверяет: если вершина не изменилась с момента снятия "снимка", она заменяется на новый узел. Если вершина изменилась, горутина повторяет попытку.

## Запуск демо-кода
В проекте есть встроенная команда для проверки стека под нагрузкой:

go run cmd/main.go

Программа запустит 100 горутин, каждая из которых выполнит по 1000 операций Push, после чего проверит целостность данных с помощью Pop.

```bash
go run cmd/stack.go
Запуск: 100 горутин по 1000 операций каждая...
Все операции завершены успешно!
Результат: Хорошо! Данные не потеряны.
```
